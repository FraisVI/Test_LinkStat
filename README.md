# LinkStat

Laravel-приложение для тестового задания: сервис генерации коротких ссылок со статистикой переходов и личным кабинетом на FilamentPHP.

## Возможности

- регистрация и вход пользователя;
- создание коротких ссылок;
- редирект по короткой ссылке на оригинальный URL;
- фиксация каждого перехода;
- хранение IP-адреса и даты перехода;
- список ссылок текущего пользователя;
- удаление ссылок;
- просмотр статистики по каждой ссылке;
- фильтрация переходов по IP-адресу и дате;
- сброс статистики отдельной ссылки;
- сводка по количеству ссылок и кликов в кабинете.

## Стек

- PHP 8.2
- Laravel 11
- FilamentPHP 3
- Nginx
- MySQL 8.4
- Docker Compose

## Запуск через Docker

Собрать и запустить контейнеры:

```bash
docker-compose up -d --build
```

Установить зависимости внутри контейнера, если папки `vendor/` нет:

```bash
docker-compose exec -T --user www-data app composer install
```

Сгенерировать ключ приложения, если он еще не задан:

```bash
docker-compose exec -T --user www-data app php artisan key:generate
```

Выполнить миграции:

```bash
docker-compose exec -T --user www-data app php artisan migrate
```

Приложение доступно по:

```text
http://localhost:8080
```

Личный кабинет доступен по:

```text
http://localhost:8080/cabinet
```

## Работа с кабинетом

1. Зарегистрироваться на странице `/cabinet/register`.
2. Войти в кабинет.
3. Открыть раздел `Ссылки`.
4. Создать короткую ссылку, указав оригинальный URL.
5. Скопировать короткую ссылку из таблицы.
6. Перейти по короткой ссылке.
7. Открыть `Статистика`, чтобы посмотреть IP-адреса и даты переходов.

## База данных

Настройки базы данных для Docker уже указаны в `.env.example`:

```text
DB_CONNECTION=mysql
DB_HOST=mysql
DB_PORT=3306
DB_DATABASE=linkstat
DB_USERNAME=linkstat
DB_PASSWORD=secret
```

Порт MySQL на хосте:

```text
3307
```

## Docker

Текущая Docker-конфигурация предназначена для разработки. Код проекта подключается в контейнер через локальный volume:

```yaml
./:/var/www/html
```

На Windows такой режим может работать медленнее из-за bind mount. Для более быстрой работы проект лучше держать внутри файловой системы WSL.
